## 表格对比

| 类型| 位数| 典型范围 / 精度| 读取格式说明| 输出格式说明| 备注|
| ---| --- | --- | ---| ---| --- |
| `char`| 8位 | -128~127（有符号）或 0~255（无符号）| `%c`（字符）或 `%hhd`（有符号整数） | `%c` 或 `%hhd` | 多用于字符，读写用 `%c`|
| `short`| 16位 | -32,768 到 32,767 | `%hd` | `%hd` | |
| `int`| 32位 | ±2×10^9 | `%d` | `%d` | 最常用整数类型 |
| `long`| 32位或64位（平台相关）| 32位系统同`int`，64位系统一般是 ±9×10^{18} | `%ld`| `%ld` | Linux 64位上通常是64位 |
| `long long`| 64位 | ±9×10^{18} | `%lld` | `%lld` | 64位整数，适合存放超大整数 |
| `float`| 32位 | 单精度浮点数，约6\~7位十进制有效数字 | `%f` 或 `%g` | `%f` 或 `%g` | 读写浮点时注意精度和范围 |
| `double`| 64位 | 双精度浮点数，约15\~16位十进制有效数字 | `%lf` | `%lf` | 读取时必须用 `%lf`，printf 中 `%f` 也支持 |
| `long double`| 80/96/128位（平台相关）| 更高精度浮点数，依实现不同 | `%Lf` | `%Lf` | 精度更高，非所有编译器完全支持 |

## 一些用法

```c
# scanf的格式指定符中不能指定精度
#scanf("%d",&n);

# 保留两位小数
printf("%.2d",n);

# %g表示自动选择 %f（普通浮点）或 %e（科学计数法）中的一种
# 保留6位有效数字
printf("%.6g",n);

# 宽度为2，不足补零
printf("%02d",n);
```

## 注意事项

### 前导空白符干扰：
指在读取输入数据时，输入流中可能存在的多余空白字符（空格、制表符、换行符等）影响程序正确读取所需数据的问题。
#### 在 C 语言里，空白字符通常包括：

    空格 ' '
    制表符 '\t'
    换行符 '\n'
    垂直制表符 '\v'
    回车符 '\r'
    换页符 '\f'

这些都是被认为是“空白字符”。

| 项目    | `scanf` 中的空白字符       | `printf` 中的空白字符       |
| ----- | -------------------- | --------------------- |
| 作用    | 跳过输入中的所有空白字符         | 原样输出一个空格字符            |
| 可跳过内容 | 空格、换行、制表、回车等         | 无，空格只是被输出             |
| 示例    | `scanf("%d %d",...)` | `printf("%d %d",...)` |


#### scanf 函数处理空白符的规则：

1. 格式字符串中的空白符
```c
scanf("%d %d");   // 空格匹配任意数量空白符
```
- 格式字符串中的空白符（`空格`、`\t`、`\n`）会匹配输入流中的任意数量连续空白符
- 包括零个空白符（即**不需要实际输入空白符**）

2. 非空白符格式说明符
```c
scanf("%d");      // 自动跳过前导空白符
```
- 对于 `%d`, `%f`, `%lf` 等数值类型
- 对于 `%s` 字符串类型
- 这些格式说明符会**自动跳过输入前的所有空白符**

3. 字符格式说明符
```c
scanf("%c");      // 读取下一个字符，包括空白符
```
- `%c` 不会跳过任何空白符
- 会读取输入流中的下一个字符，即使它是空白符
具体可以查看[映射排序](baseAndIO\mappingSorting\mappingSorting.c)
