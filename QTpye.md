# 数学建模类型
1. 数学建模是指：
   面对一个现实或抽象的问题，先抽象出数学模型（用公式、函数、方程表示问题），再用数学方法分析、求解，最后将结果转化成实际解。
2. 这种一般涉及到方程组等数学问题，需要建立一个统一的数学模型，然后用数学方法解决。
3. 特点：
| 特征        | 说明                       |
| --------- | ------------------------ |
| 先抽象再求解    | 不是一开始就写代码，而是构造方程或模型      |
| 用到较多数学知识  | 比如线性规划、微积分、概率、图论、最优化     |
| 通常问题更开放   | 可能有多个模型、参数，需你自己定义变量与目标   |
| 常用于现实问题建模 | 比如“资源分配”、“运输优化”、“疫情传播预测” |

# 模拟题
1. 模拟是指：
把题目中描述的规则、过程一步步原样实现出来，**不需要复杂数学公式推导**，而是直接照着题意进行处理。
2. 特点：
| 特征         | 说明              |
| ---------- | --------------- |
| 过程可操作      | 按照题目描述的步骤逐步执行   |
| 往往与题目语言强绑定 | 一句一句“翻译”题意成代码   |
| 数据量小-中     | 不强调效率优化，但强调逻辑正确 |
| 数学公式少或简单   | 有四则运算、取整、循环判断即可 |

# 枚举类型
1. 什么是“枚举”？
枚举就是“把所有可能的情况一个个试一遍”。
2. 枚举的思路： 
换句话说，程序穷举所有合法选项，再从中找到满足条件的最优解、最大值、最小值或特定答案。
| 特征                   | 说明                              |
| -------------------- | ------------------------------- |
|  **输入规模小**          | 枚举题通常数据范围不大，能“遍历所有可能”而不超时       |
|  **问题要求最值或满足条件的情况** | 例如“找最大值/最小值”、“找某种条件下最先/最后出现的情况” |
|  **没有复杂的状态变化**      | 枚举题往往逻辑清晰，没有动态规划或贪心那种复杂转移       |
|  **暴力可行，优化不是必须的**   | 即使不用高效算法，直接试也能通过所有测试点           |
